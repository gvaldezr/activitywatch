# ActivityWatch Watcher Development Guide

Standardized workflow and patterns for developing new watchers or modifying existing ones in ActivityWatch.

## Watcher Architecture

A **watcher** is a standalone process that:
1. Collects data from the system (e.g., active window, AFK status, keyboard input)
2. Transforms it into structured **Events**
3. Sends events to **aw-server** via **aw-client**

### Core Watchers (Production)
- **aw-watcher-window**: Active application and window title (platform-specific implementations)
- **aw-watcher-afk**: Away-from-keyboard detection via input events
- **aw-watcher-input**: Keyboard/mouse activity (optional extra)

### Watcher Lifecycle

```
Data Collection → Event Creation → Batching/Bucketing → aw-client.send() → aw-server (REST API)
```

## Creating a New Watcher

### Project Structure
```
aw-watcher-myname/
├── Makefile              # Standard build targets
├── pyproject.toml        # Poetry dependencies
├── aw_watcher_myname/
│   ├── __init__.py
│   └── watcher.py        # Main watcher logic
├── tests/
│   └── test_watcher.py   # pytest tests
```

### Step 1: Define Dependencies (pyproject.toml)

```toml
[tool.poetry.dependencies]
python = "^3.8"
aw-core = "*"              # For Event, Bucket data models
aw-client = "*"            # For sending data to server
# Add platform-specific or data collection libraries here
```

### Step 2: Implement Core Watcher Logic

```python
# aw_watcher_myname/watcher.py
from aw_core.models import Event
from aw_client import ActivityWatchClient

class MyWatcher:
    def __init__(self, client: ActivityWatchClient):
        self.client = client
        self.bucket_id = "aw-watcher-myname"
        
    def run(self):
        """Main watcher loop"""
        self.client.create_bucket(self.bucket_id, event_type="myevent")
        
        while True:
            # Collect data
            data = self.collect_data()
            
            # Create event
            event = Event(
                timestamp=datetime.now(timezone.utc),
                duration=timedelta(seconds=data['duration']),
                data={"key": data['value']}
            )
            
            # Send to server
            self.client.send_event(self.bucket_id, event)
            time.sleep(1)  # Polling interval
    
    def collect_data(self):
        """Override this with actual data collection logic"""
        pass
```

### Step 3: Add CLI Entry Point

```python
# aw_watcher_myname/__main__.py
import logging
from aw_client import ActivityWatchClient
from .watcher import MyWatcher

def main():
    client = ActivityWatchClient("aw-watcher-myname", testing=False)
    watcher = MyWatcher(client)
    watcher.run()

if __name__ == "__main__":
    main()
```

## Integration with ActivityWatch

### Bucket Management
- **Bucket ID**: Uniquely identifies the data source (e.g., "aw-watcher-window")
- **Event type**: Categorizes events (e.g., "currentwindow", "afk")
- Always use `client.create_bucket()` before sending events

```python
client.create_bucket(
    bucket_id="aw-watcher-myname",
    event_type="myevent_type",
    hostname="myhost"
)
```

### Event Schema
```python
Event(
    timestamp=datetime.now(timezone.utc),  # Required
    duration=timedelta(seconds=1),          # Required
    data={                                  # Custom dict
        "category": "value",
        "details": {...}
    }
)
```

### Error Handling
```python
try:
    self.client.send_event(self.bucket_id, event)
except Exception as e:
    logger.error(f"Failed to send event: {e}")
    # Implement retry logic or queuing if needed
```

## Common Patterns

### Polling-Based Watcher
```python
while True:
    state = self.get_current_state()
    event = Event(..., data=state)
    self.client.send_event(self.bucket_id, event)
    time.sleep(self.poll_interval)
```

### Event-Driven Watcher
```python
def listener():
    for event in self.system_listener.listen():
        aw_event = Event(..., data=event)
        self.client.send_event(self.bucket_id, aw_event)
```

### Batching Events
```python
events = []
while True:
    state = self.get_current_state()
    events.append(Event(...))
    if len(events) >= batch_size:
        self.client.send_events(self.bucket_id, events)
        events = []
```

## Testing Strategy

1. **Unit tests**: Test data collection, event creation, transformations
2. **Integration tests**: Test communication with aw-server (may use testing mode)
3. **Platform tests**: Verify platform-specific code works on target OS
4. **Manual testing**: Run watcher, verify events appear in ActivityWatch Web UI

```bash
# Run tests
make test

# Run with debug logging
RUST_LOG=debug poetry run python -m aw_watcher_myname
```

## Distribution

### Packaging for Release
- Update version in `pyproject.toml`
- Follow [Conventional Commits](https://www.conventionalcommits.org/)
- Tests must pass: `make test && make lint && make typecheck`
- Pyinstaller hooks handled at monorepo level

### Adding to Monorepo
1. Create module as git submodule under root
2. Add to `SUBMODULES` list in root Makefile (optional)
3. Ensure Makefile has `build`, `test`, `lint`, `typecheck` targets
4. Document in README and AGENTS.md

## Best Practices

- **Low overhead**: Minimize CPU/memory usage; watchers run continuously
- **Graceful degradation**: Handle platform differences; skip unavailable features
- **Error recovery**: Reconnect to server automatically if connection fails
- **Configuration**: Support environment variables or config files for settings
- **Documentation**: Document data format, bucket schema, privacy implications
- **Testing**: Aim for >80% code coverage; test edge cases
